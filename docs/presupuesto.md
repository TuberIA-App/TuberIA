# Presupuesto Económico - TuberIA

**Documento**: Criterio 2f - Presupuesto económico basado en tiempo invertido  
**Período**: 6 sprints (01 Diciembre 2025 - 11 Enero 2026)  
**Equipo**: 3 desarrolladores Junior (20 €/hora)

---

## 1. Metodología

### 1.1. Estimación
- **Técnica**: Planning Poker con secuencia Fibonacci (0.5, 1, 2, 3, 5, 8, 13, 21 horas)
- **Proceso**: Durante Sprint Planning, estimación simultánea por cada historia. Discusión si discrepancia >3 puntos. Consenso o mediana.
- **Campo**: GitHub Projects > campo personalizado "Estimación (horas)" por issue.

### 1.2. Medición de tiempo real
- **Herramienta**: Toggl Track (plan gratuito)
- **Flujo**:
  1. Abrir issue en GitHub
  2. Iniciar timer en Toggl Track
  3. Trabajar en la tarea
  4. Añadir comentario de progreso en el issue
  5. Detener timer al finalizar/pausar
- **Campo**: GitHub Projects > campo personalizado "Horas Reales" por issue.

### 1.3. Tarifa horaria
- **Junior Developer**: 20 €/hora (equipo académico, 0-2 años experiencia)
